# Starter Templates

This folder contains minimal starter code for different phases of the lab. Use these if participants join late or need to skip ahead.

## Available Starters

| Starter | Lessons | Description |
|---------|---------|-------------|
| [00-empty](00-empty/) | 00-02 | Empty project with just instructions |
| [03-scaffold](03-scaffold/) | 03 | Project structure, package.json files only |
| [05-backend](05-backend/) | 04-05 | Working backend with API and Foundry |
| [08-fullstack](08-fullstack/) | 06-08 | Full app without polish |
| [11-complete](11-complete/) | 09-11 | Feature-complete, needs polish |

## How to Use

1. Copy the starter folder to participant's workspace
2. Run `npm install` in root, api/, and web/ folders
3. Continue from the indicated lesson

## For Instructors

- For late arrivals, start them at the nearest checkpoint
- Each starter includes a `START_HERE.md` with context
- Starters are intentionally minimal - they should build the rest
